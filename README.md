# AI Fact Validator

AI Fact Validator is a web-based application built to verify factual claims and help detect misinformation. The system uses AI models, live web search, and semantic vector search to check whether a claim is true or false.

One of the main goals of this project is to reduce repeated verification work. If a similar claim has already been verified earlier, the system can directly return the stored result instead of running the full AI verification process again.

---

# Features

- Real-time fact verification using AI
- Live web evidence collection
- Detection of fake or verified claims
- Confidence score generation
- Semantic duplicate detection
- Contradiction detection
- Human review flagging
- Storage of verified claims in Pinecone vector database
- Smart suggestion system while typing
- Direct retrieval of pre-verified results
- Faster response time using cached verified claims

---

# How the System Works

## Claim Verification Flow

When a user enters a claim for verification:

```text
User Query
   ↓
Alpha Agent
   ↓
Web Search + AI Analysis
   ↓
Structured Verification Result
   ↓
Frontend Response
   ↓
Beta Agent
   ↓
Vector Database Comparison
   ↓
Insert / Reject / Human Review
```

### Alpha Agent

The Alpha Agent handles the actual fact verification.

Its responsibilities:

- identify whether the query is factual or casual
- collect relevant information from web search
- analyze the available evidence
- generate the final verification result

Example response:

```json
{
  "claim": "Is Earth flat?",
  "verification_status": "fake",
  "summary": "There is no scientific evidence supporting the claim that Earth is flat.",
  "sources": ["https://www.nasa.gov"],
  "confidence": 98
}
```

---

### Beta Agent

The Beta Agent manages storage and duplicate detection.

Its responsibilities:

- generate embeddings for verified claims
- search Pinecone vector database
- check for semantically similar claims
- detect contradictions
- decide whether the claim should be stored

Possible decisions:

```json
{
  "flag": "insert",
  "reason": "Similar verified claim already exists."
}
```

```json
{
  "flag": "reject",
  "reason": "Similar verified claim already exists."
}
```

```json
{
  "flag": "human_review",
  "reason": "Similar verified claim already exists."
}
```

# Suggestion System

The project also includes a smart suggestion feature.

As the user starts typing a query, the system performs semantic search in the vector database and shows similar previously verified claims.

Flow:

```text
User Types Query
   ↓
Query Embedding
   ↓
Pinecone Similarity Search
   ↓
Matching Verified Claims
   ↓
Suggestion Dropdown
```

If the user clicks a suggestion, the system directly returns the stored verified result.

This means:

- no AI verification call
- no web search
- faster response
- reduced API cost

This improves overall performance and user experience.

## Save Verified Claim to Database

**POST**
```bash
/db/save
```

This endpoint stores the verified response generated by the Alpha Agent into the vector database after semantic validation by the Beta Agent.

Request:

```json
{
  "data": {
    "claim": "Is Earth flat?",
    "verification_status": "fake",
    "summary": "There is no scientific evidence supporting the claim that Earth is flat.",
    "sources": [
      "https://www.nasa.gov"
    ],
    "confidence": 98
  }
}
```

Possible Response:

```json
{
  "flag": "insert",
  "reason": "New claim detected and stored successfully."
}
```

```json
{
  "flag": "reject",
  "reason": "Similar verified claim already exists."
}
```

```json
{
  "flag": "human_review",
  "reason": "Contradictory claim found in database."
}
```


# Purpose of the Project

The purpose of this project is to demonstrate how AI agents, live web verification, and semantic vector databases can be combined to build a practical misinformation detection system.

Instead of verifying the same claims repeatedly, the system intelligently stores verified claims and reuses them when similar queries appear in the future.
This makes the system faster, more efficient, and more scalable.
